# Quickstart

Evaluate the full Codemagic Patch service on your machine before provisioning domains or OAuth. The **local evaluation stack** runs the real server, worker, Postgres, MinIO, and dashboard, with sign-in replaced by a local one-click login. Then use the bundled [on-device demo](https://github.com/codemagic-ci-cd/codemagic-patch/tree/main/examples/on-device-demo) to watch an OTA update apply on a simulator or emulator.

## What you need

- [Docker](https://docs.docker.com/get-docker/) with Compose v2
- Node.js `≥ 22.20` (for the CLI install)
- Yarn via [Corepack](https://nodejs.org/api/corepack.html) (`corepack enable`)
- For the demo app: an **iOS Simulator** (macOS + Xcode) and/or an **Android emulator** (`adb` on `PATH`)

## 1. Start the stack

```bash
git clone https://github.com/codemagic-ci-cd/codemagic-patch.git
cd codemagic-patch
./scripts/local-eval/up.sh
```

The script builds and starts `docker-compose.dev.yml`, installs the `cmpatch` CLI globally, seeds a demo app, and prints a ready banner. The first run can take a few minutes while images build.

When it finishes you get:

- **Dashboard** — [http://localhost:8080](http://localhost:8080) (sign in as `local-admin@example.com`, prefilled, one click)
- **API** — [http://localhost:3000](http://localhost:3000)
- Seeded fixtures for `demo-app`, including the deployments the on-device demo uses

:::warning
Evaluation only, not a deployment. Authentication is disabled and all ports bind to localhost. For production, follow [Self-host](/docs/setup/self-host).
:::

## 2. Sign in with the CLI

```bash
cmpatch login --server-url http://localhost:3000
```

The local stack approves the device flow instantly.

## 3. Watch an update apply (on-device demo)

[`examples/on-device-demo`](https://github.com/codemagic-ci-cd/codemagic-patch/tree/main/examples/on-device-demo) is a minimal React Native app preconfigured against this stack. It shows a blue **v1** banner. You change one line, publish, relaunch, and the banner flips to green **v2**. That flip is Patch replacing the JS bundle.

One-time setup and install (from the demo directory):

```bash
cd examples/on-device-demo
yarn install
yarn demo:setup:ios   # iOS only — installs pods

yarn demo:ios         # iOS Simulator
# or
yarn demo:android     # Android emulator (also sets up adb reverse for localhost)
```

Both commands build **Release** with `--no-packager`, so the app boots from its embedded bundle (the one OTA replaces). On launch you should see the blue **v1** banner and *"Up to date — you are running v1."*

Edit `App.tsx` and change the marked line:

```ts
const APP_VERSION = 'v1';   // change to 'v2'
```

Publish from the same directory:

```bash
# iOS
cmpatch release-react \
  --server-url http://localhost:3000 \
  --app demo-app --deployment staging-ios \
  --platform ios

# Android
cmpatch release-react \
  --server-url http://localhost:3000 \
  --app demo-app --deployment staging-android \
  --platform android
```

In the app, tap **Check again**. It finds the release, shows download progress, then **Update installed — Relaunch**. Tap that: the banner flips to green **v2**.

Full detail and troubleshooting: [on-device demo README](https://github.com/codemagic-ci-cd/codemagic-patch/tree/main/examples/on-device-demo).

## 4. Confirm in the dashboard (optional)

Open [http://localhost:8080](http://localhost:8080), sign in as `local-admin@example.com`, then open **demo-app** and the `staging-ios` or `staging-android` deployment. You should see the release you just published.

Without a simulator, you can still publish the bundled fixture from the repo root:

```bash
cmpatch release create \
  --server-url http://localhost:3000 \
  --app demo-app \
  --deployment staging \
  --bundle-path examples/local-dev/bundles/ios-hermes-v1.zip \
  --target-binary-version 1.0.0 \
  --fingerprint local-dev-fingerprint \
  --release-notes "Local quickstart" \
  --yes
```

## Tear down

```bash
docker compose -f docker-compose.dev.yml down -v
```

Re-print the ready banner without reinstalling the CLI:

```bash
./scripts/local-eval/up.sh --skip-cli
```

## What you just ran

| Piece | Local value |
| --- | --- |
| Dashboard | `http://localhost:8080` |
| API | `http://localhost:3000` |
| Artifact downloads | `http://localhost:9100/codemagic-patch` |
| Demo deployments | `staging-ios` / `staging-android` |
| Demo deployment keys | `dev_local_ios_deployment_key` / `dev_local_android_deployment_key` |

:::warning
The seeded API token (`cm_pat_local-dev-token-change-me-00000001`) is for local scripting only. Do not reuse it in production.
:::
