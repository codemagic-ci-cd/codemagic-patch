# Requirements

## Server host

For trying Patch on your laptop without domains or TLS, see the [Quickstart](/docs/).

Self-hosting requires:

- Docker + Docker Compose v2, and `curl`
- Public inbound access on ports 80 and 443
- Two domains with DNS A/AAAA records pointing at the host: one for the API/dashboard, one for artifact storage. They must differ:
  - API/dashboard: e.g. `updates.example.com`
  - Storage: e.g. `storage.updates.example.com`
- A GitHub OAuth App (see [Self-host](/docs/setup/self-host#prepare-a-github-oauth-app); [register one](https://github.com/settings/applications/new))
- Using Cloudflare for DNS or CDN? See [Cloudflare setup](/docs/setup/cloudflare) before install (Free plans have a nested-hostname SSL caveat)

### Machine size (default single-host stack)

The Compose install runs Caddy, the API and release worker, Postgres, and MinIO on one host.

- **RAM:** 2 GB is enough for typical use. Prefer 4 GB for large or overlapping releases, or a long on-box history of artifacts and metrics.
- **CPU:** 1 vCPU is enough for typical use.
- **Disk:** tens of GB is fine to start. Object storage grows with retained releases; Postgres grows with device metrics (`metric_event` is not auto-pruned).

## Building / using the CLI (local or CI)

- Node.js `>=22.20.0`
- Yarn `4.12.0` (via Corepack)

## React Native app

- React Native `>=0.76`, React `>=18`
