# Run locally

:::info
Use this to evaluate Patch or develop against the project on your laptop. For the shortest path, start with the [Quickstart](/docs/). This is not a production deployment: authentication is disabled and all ports bind to localhost. For a real server, see [Self-host](/docs/setup/self-host).
:::

The evaluation stack is defined in `docker-compose.dev.yml`. `./scripts/local-eval/up.sh` brings it up, installs the `cmpatch` CLI, waits for readiness, and seeds a demo app so you can publish without GitHub OAuth or hand-created resources.

## Prerequisites

- Docker + Docker Compose v2
- Node.js `≥ 22.20` and Yarn (via Corepack), for the CLI install

Pass `--skip-cli` to `up.sh` if you only want the Docker stack.

## Start the stack

From the project root:

```bash
./scripts/local-eval/up.sh
```

Equivalent Compose-only bring-up (no CLI install or ready banner):

```bash
docker compose -f docker-compose.dev.yml up --build -d
```

The stack includes Postgres, MinIO, the API + release worker (local auth entrypoint), the web dashboard (Caddy serving the built SPA on port 8080), and a one-shot seed job.

### Tear down

Stops containers and deletes dev data (Postgres + MinIO volumes):

```bash
docker compose -f docker-compose.dev.yml down -v
```

## Local endpoints

| Service | URL | Notes |
| --- | --- | --- |
| Dashboard | `http://localhost:8080` | Sign in as `local-admin@example.com` (prefilled, one click) |
| API | `http://localhost:3000` | Control plane + metrics |
| Artifact downloads | `http://localhost:9100/codemagic-patch` | Public OTA prefix on MinIO |
| MinIO console | `http://localhost:9101` | Login: `minio` / `minio12345` |
| Postgres | `localhost:55433` | DB `codemagic_patch_dev`, user/password `codemagic_patch` |

Readiness for the whole stack (dashboard proxies the server, including the DB check):

```bash
curl -fsS http://localhost:8080/health/ready
```

## Seeded fixtures

After `up`, `examples/local-dev/seed.sql` attaches these fixtures to the bootstrap team (`default-team`):

| Item | Value |
| --- | --- |
| Team | `default-team` |
| App | `demo-app` |
| Deployment | `staging` |
| Deployment key | `dev_local_deployment_key` |
| API token | `cm_pat_local-dev-token-change-me-00000001` |

The on-device demo also gets per-platform deployments (`staging-ios` / `staging-android` with `dev_local_ios_deployment_key` and `dev_local_android_deployment_key`) so iOS and Android releases do not share a manifest.

The token is for local scripting only. Prefer `cmpatch login --server-url http://localhost:3000` for interactive use; the local stack approves instantly.

## Publish a test release

```bash
cmpatch login --server-url http://localhost:3000

cmpatch release create \
  --server-url http://localhost:3000 \
  --app demo-app \
  --deployment staging \
  --bundle-path examples/local-dev/bundles/ios-hermes-v1.zip \
  --target-binary-version 1.0.0 \
  --fingerprint local-dev-fingerprint \
  --release-notes "Local smoke test" \
  --yes
```

From a React Native project root (with the SDK integrated):

```bash
cmpatch init \
  --server-url http://localhost:3000 \
  --ios-app demo-app \
  --deployment staging \
  --yes

cmpatch release-react --platform ios --deployment staging --release-notes "Local smoke test" --yes
```

## Point your app at localhost

Use these values in native config or the Expo plugin:

| Config key | Local value |
| --- | --- |
| `CodemagicPatchApiUrl` | `http://localhost:3000` |
| `CodemagicPatchDownloadBaseUrl` | `http://localhost:9100/codemagic-patch` |
| `CodemagicPatchDeploymentKey` | `dev_local_deployment_key` |

:::warning
Physical devices and Android emulators cannot reach your laptop's `localhost` the same way the iOS Simulator does. Use your machine's LAN IP (or `10.0.2.2` on the Android emulator) for both URLs, and ensure `PUBLIC_BASE_URL` in the Compose file matches what the device can fetch. The default Compose file uses `http://localhost:9100/codemagic-patch`, which works for iOS Simulator on the same Mac. The [on-device demo](https://github.com/codemagic-ci-cd/codemagic-patch/tree/main/examples/on-device-demo) uses `adb reverse` for the Android emulator.
:::

## On-device demo

`examples/on-device-demo/` is a minimal React Native app preconfigured against this stack. With the evaluation stack up, you can build for simulator or emulator, publish a one-line JS change, and watch the OTA apply. See that directory's README for the exact commands.

## Local vs self-host

| | Local evaluation stack | Self-host installer |
| --- | --- | --- |
| Purpose | Try Patch / develop on your machine | Production-like deployment |
| Bring-up | `./scripts/local-eval/up.sh` | `scripts/selfhost/install.sh` |
| HTTPS / domains | No (localhost only) | Yes (Let's Encrypt) |
| Dashboard | Served on `:8080` | Served by Caddy on your API domain |
| Auth | Local one-click login + instant CLI approval | GitHub OAuth |
| Object storage | MinIO on `localhost:9100` | MinIO behind storage domain |
